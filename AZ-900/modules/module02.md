# Module 2 - Describe the Benefits of Using Cloud Services

Status: ✅ Completed

## Exam Objective

Differentiate the major cloud benefit terms and map each one to a practical outcome.

## Core Definitions

- Availability: Probability that a service is accessible when needed.
- Uptime: Percentage of time a system is running, regardless of app usability.
- Reliability: Ability of a system to recover and continue operating.
- Resiliency: Ability to return to normal operation after faults.
- Scalability: Ability to handle increased load by adding capacity.
- Elasticity: Ability to automatically scale up and down with demand changes.
- Predictability: Ability to estimate performance and cost before deployment.
- Agility: Speed of provisioning and changing resources.
- Governance: Policies and controls that guide secure and compliant cloud operations.
- Manageability: Ability to operate and monitor resources effectively using tools and automation.

## High-Value Comparisons

- Availability vs uptime: Uptime can be high while app is still unusable; availability is user-visible service access.
- Scalability vs elasticity: Scalability is growth capability, elasticity is automatic expansion and contraction.

## Security and Governance Benefits

- Built-in security controls are available by default in many Azure services.
- Policy-driven governance improves consistency and reduces misconfiguration risk.
- Standardized templates and automation improve compliance posture.
- Provider scale improves baseline security investment compared to many on-prem environments.

## Manageability in Azure

- Common management interfaces: Azure Portal, Azure CLI, Azure PowerShell, ARM/Bicep templates.
- Operational capabilities include monitoring, alerting, auto-healing patterns, and consistent deployment.

## Common Exam Traps

- "Scales quickly and automatically" points to elasticity, not just scalability.
- "Consistent policy enforcement" points to governance.
- "Known expected cost/performance" points to predictability.
