# Module 3 - Describe Cloud Service Types

Status: ✅ Completed

## Exam Objective

IaaS, PaaS, SaaS and selecting the right model by responsibility and control.

## Core Definitions

- IaaS (Infrastructure as a Service): Cloud model where provider supplies compute, storage, and networking; customer manages OS, middleware, app, and data.
- PaaS (Platform as a Service): Provider also manages OS/runtime/platform; customer focuses on application code and data.
- SaaS (Software as a Service): Provider delivers ready-to-use software; customer configures and uses it.

## Responsibility Summary

- IaaS: Highest customer control, highest customer management effort.
- PaaS: Balanced control and productivity.
- SaaS: Lowest management effort, least infrastructure control.

## Typical Azure-Aligned Examples

- IaaS: Azure Virtual Machines.
- PaaS: Azure App Service, Azure SQL Database.
- SaaS: Microsoft 365 and similar ready-to-use applications.

## Use Case Mapping

- IaaS:
	- Lift-and-shift workloads.
	- Custom OS-level configuration.
	- Legacy app dependencies.
- PaaS:
	- Fast app development and deployment.
	- Auto-scaling web/API workloads.
	- Reduced platform maintenance.
- SaaS:
	- Email, collaboration, CRM, ticketing.
	- Fast business adoption with minimal IT setup.

## Common Exam Traps

- If question mentions patching OS yourself, it leans IaaS.
- If question mentions coding app logic without managing servers, it leans PaaS.
- If question mentions using a complete app immediately, it leans SaaS.
